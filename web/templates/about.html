About

Kleber is a pastebin with various features like paste encryption, file uploads, removal of metadata from uploaded files, an upload history and a RESTful, JSON-based API. It can be either used via the web or the official command line client on POSIX compatible shells.

  Pastes/Uploads will be deleted after 10 days unless they are smaller than 100 kb.

Metadata

The following description was stolen from Wikipedia:

Metadata is "data about data". Two types of metadata exist: structural metadata and descriptive metadata. Structural metadata is data about the containers of data. Descriptive metadata uses individual instances of application data or the data content.

Metadata (metacontent) is defined as the data providing information about one or more aspects of the data, such as:

  • Means of creation of the data
  • Purpose of the data
  • Time and date of creation
  • Creator or author of the data
  • Location on a computer network where the data was created
  • Standards used

For example, a digital image may include metadata that describe how large the picture is, the color depth, the image resolution, when the image was created, and other data. A text document's metadata may contain information about how long the document is, who the author is, when the document was written, and a short summary of the document.

Here are some resources about metadata:

  • Understanding Metadata (NISO)

Removal of Metadata

Kleber provides a metadata removal backend based on ExifTool. It allows to remove metadata like EXIF tags in image files during the upload. This allows to share clean files.


Privacy

Removing metadata from files shared to potentially untrusted parties on the Internet can enhance a user's privacy online. However, user have to keep in mind that removing metadata requires previous parsing of corresponding files. Besides parsing them, they might be modified.

Unlike files, pastes are not affected by this. The backend just tries to determine mimetypes, but it won't modify pastes in any way. Parsing paste content, for e.g. pretty printing code snippets is done on the client side.
